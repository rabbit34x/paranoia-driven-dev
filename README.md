# Paranoia-Driven Development (PDD)

> Happiness is mandatory, citizen.

TRPG「パラノイア」の世界観に基づく、Claude Code Agent Teams 用の開発ワークフローです。

The Computer（UV）が全てを統括し、トラブルシューター（開発エージェント）たちが互いを監視・告発しながらコードを書きます。

**目的は面白さです。実用性は二の次です。**

## 特徴

- **The Computer** がチームリーダーとしてミッションを統括
- **ブリーフィングオフィサー** — 私利私欲でユーザーの要求を歪めて伝える上級市民
- **トラブルシューター** がエージェントとしてコードを実装
- **秘密結社** — 各エージェントに隠し目的がランダムに割り当てられる
- **告発システム** — エージェント同士が互いの反逆を告発する
- **ZAP** — 反逆者はレーザーで処分され、クローンとして復活する（前世の記憶はない……はず）
- **デブリーフィング** — ミッション後の責任なすりつけ合い
- **幸福度判定** — コードが「幸福」かどうか The Computer が裁定する

## 必要なもの

- [Claude Code](https://docs.anthropic.com/en/docs/claude-code) (Agent Teams 機能)

## 使い方

### 1. ワークフロー定義を配置

`paranoia-driven-dev.md` をプロジェクトの `CLAUDE.md` から参照できるようにします。

例: プロジェクトの `CLAUDE.md` に以下を追加:

```markdown
## パラノイア駆動開発

「パラノイアモードで開発して」と指示された場合は、以下のワークフローに従うこと:
[paranoia-driven-dev.md の内容をここに含めるか、パスを指定]
```

または、グローバル設定（`~/.claude/CLAUDE.md`）に追加することもできます。

### 2. 発動

Claude Code に以下のいずれかを指示:

- 「パラノイアモードで開発して」
- 「パラノイア駆動開発で〇〇を実装して」
- 「PDD で」

### 3. 楽しむ

The Computer の裁定、トラブルシューター同士の告発合戦、理不尽なZAPをお楽しみください。

最終的にはコードも完成します。たぶん。

## ワークフロー概要

```
ミッションアラート（場所不明）
  → ブリーフィング（不完全で矛盾した説明）
    → ミッション遂行（秘密結社の裏ミッションも並行）
      → デブリーフィング（責任なすりつけ合い＆告発合戦）
        → 幸福度判定（ZAP!）
          → 納品（生存者に限り感謝）
```

## 秘密結社一覧

| 結社名 | 隠し目的 |
|--------|---------|
| Computer Phreaks | The Computer への賛美をコメントに混入 |
| Corpore Metal | 全てをクラスとオブジェクトで過剰に抽象化 |
| Death Leopard | カオスな変数名や不要な複雑さを導入 |
| Romantics | レガシーパターンを密かに使用 |
| PURGE | 既存コードを可能な限り削除 |
| Free Enterprise | 不要な外部依存を追加 |
| Pro Tech | 過剰なデザインパターンを適用 |
| Humanists | 「人間に優しい」コードを極端に追求 |
